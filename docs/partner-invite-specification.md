# 代表者/パートナー機能 仕様書

## 概要

会員サイト（SMG経営塾）に入会した代表者（メイン会員）は、1名だけ「パートナー」として別のユーザーを招待し、同じ会員サイトを利用させることができる仕組み。

## 用語定義

| 用語 | 説明 |
|------|------|
| **代表者** | 会員サイトに正式入会した本人。課金・契約の主体。 |
| **パートナー** | 代表者が招待した1名のユーザー。代表者の権限に準じてサイトを利用可能。 |

## 仕様詳細

### 招待ルール
- 代表者1名につき、パートナーは **1名のみ** 招待可能
- パートナーの追加上限は1名（複数招待不可）

### パートナーの権限
- 会員サイトの基本機能を利用可能（詳細は今後定義）
- 代表者と同等の閲覧権限を持つ想定

### DM一覧での表記（将来実装候補）
- DMスレッド一覧に「代表者」「パートナー」のラベルを表示する要望あり
- 現時点では **未実装**（将来的な優先度：低）

### データモデル（検討中）
- `mst_user` テーブルに `invited_by` (UUID, nullable) カラムを追加
  - `invited_by IS NULL` → 代表者
  - `invited_by IS NOT NULL` → パートナー（値は代表者のuser_id）
- もしくは `trn_invite` テーブルで管理
